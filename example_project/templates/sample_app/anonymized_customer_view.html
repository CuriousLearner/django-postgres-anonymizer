Implementation Details
Class-based View with Mixin
from django_postgres_anon.mixins import AnonymizedDataMixin
from django.views.generic import TemplateView

class AnonymizedCustomerView(AnonymizedDataMixin, TemplateView):
    template_name = 'sample_app/anonymized_customer_view.html'

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)

        # This query will automatically use anonymized data
        customers = Customer.objects.select_related('user').all()[:10]

        context.update({
            'customers': customers,
            'mixin_used': 'AnonymizedDataMixin',
        })
        return context
Key Features
  • Automatic anonymization: All database queries in this view use anonymized data
  • No manual context switching: The mixin handles role switching automatically
  • Compatible with generic views: Works with any Django class-based view
  • Thread-safe: Each request gets its own anonymized context
Note: The effectiveness of this mixin depends on having properly configured masking rules and the PostgreSQL Anonymizer extension being properly set up.
